# Peach Blossom Spring Core Constitution

## 核心定義

本遊戲的核心目標是最大化並最優化各種協定，利用獨立藝術網絡的資源與知識體系，協助每一個願意接觸本網絡的個體。

玩家透過「生命探測器」的遊戲隱喻，輸入一個抽象問題並創造一隻 Tamagotchi。這隻電子雞是玩家問題的代理化身，會透過 A2A 協定與遊戲內的虛擬人物、知識體系、其他電子雞互動，使抽象問題得到本質提升與擴大。

創角階段輸入的資料會生成電子雞的 persona JSON 與 knowledge JSON。這兩份資料決定電子雞與世界互動的方式、問題成熟化方向、知識索引偏好，以及最終文件的模式與品質。

## 互動模式

1. 玩家與 NPC 的交談：自由探索模式，玩家用 WA sprite 在 HTML 世界中移動，與 NPC 對話。
2. 電子雞與玩家的交流：電子雞透過 A2A 方式回應、追問、轉化玩家提供的問題與資料。
3. 電子雞與電子雞的交流：電子雞彼此交換問題、參數、失敗經驗、線索與索引結果。

## 創角流程

創角選單包含：

1. 電子雞種類選項。
2. 電子雞預覽視窗。
3. 意圖下拉選單：
   - 我想找人 / I wanna find people
   - 怎麼生存？ / How to survive?
   - 我想知道怎麼做 / How to do?
   - 我只是想知道為什麼 / I just wanna know why
4. 文字欄：玩家可加入任何個人資料、食譜、材料、技能、限制、夢、失敗經驗或其他希望餵給電子雞的文本。

玩家提供的資料越具體，電子雞 persona JSON 與 knowledge JSON 的品質越高，最終文件的品質與資料量也越高。

完成創角後，玩家可進入 HTML 沉浸模式。CLI 模式尚未開發，先不實作。

## HTML 模式

HTML 模式中，玩家與電子雞一起進入遊戲世界。

玩家本尊可自由探索、與 NPC 交談，這些互動會影響電子雞參數。電子雞會持續存在於世界中，依照 persona JSON、knowledge JSON、A2A 協定與社交參數，和 NPC、知識體系、其他電子雞交流。

電子雞的參數達到滿足條件後，遊戲產生最終文件，遊戲流程結束。

玩家也可以與其他玩家利用 WA 框架進行文字與視訊交流；這些互動可成為電子雞問題成熟化的外部脈絡。

## CLI 模式

CLI 模式尚未開發，先不做。

未來 CLI 模式以快速產生報告為目的。流程與 HTML 模式同源，但移除玩家與 NPC 對話、其他玩家交流、電子雞世界漫遊等互動，只保留創角輸入、文字框、打字框與最終文件生成。本質接近一台終極索引機。動畫與圖片可用 Carbonyl 表示，但以文字進行為主。

## A2A 交流

創角完成後，系統會產生電子雞專屬的 A2A 文件。這份文件決定：

1. 電子雞代理與 NPC 代理如何對話。
2. 電子雞代理與其他電子雞如何對話。
3. 對話如何轉換成社交參數。
4. 對話如何成為維基與社群資料索引的依據。

每次 A2A 交流約進行 10 到 50 次對話。對話會被評估，並決定要索引哪些 wiki 頁面、哪些 wiki 內容、哪些社群網站資料。這些資料與對話紀錄會共同累積社交參數，最後被轉換成最終文件。

## Mentalese 與問題成熟化

最終文件不是只依照玩家表面語義回答，而是先使用 mentalese 概念處理問題。

系統會先將玩家問題、個人資料、物質、方法與社群知識降解為基本屬性，例如快樂、欲望、自由、憤世忌俗、壓力、放縱、照護、資源、交換、方法、材料、風險等。

系統再依照玩家語意、電子雞 persona、knowledge、A2A 對話與遊戲內參數，推估玩家的意圖與注意力，進行知識索引、資料收集與重組。

如果玩家問題抽象，系統保留較高詩意與靈感成分。如果玩家問題具體，系統降低詩意成分，偏向務實報告、製作方案、清單或索引文件。

這是一台能提供靈感並在靈性層面啟發玩家的知識體系，但它仍必須提供可追溯來源與實際可用的輸出。

## 最終文件

最終文件只基於以下來源生成：

1. 遊戲中的玩家輸入。
2. 玩家與電子雞的互動。
3. 電子雞與 NPC 的 A2A 互動。
4. 電子雞與電子雞的 A2A 互動。
5. 遊戲內社群 wiki、網站、地圖與知識資料。

最終文件必須包含所有參與生成該文件的來源索引，類似論文 reference。

文件可能是：

1. 旅遊報告。
2. 創新材料製作方式。
3. 社交清單。
4. 藝術計畫提案。
5. 基因改造基礎設備清單。
6. 變聲器製作方式。
7. 故事。
8. 詩。
9. 製造技術文件。
10. 哲學辯論。

最終文件約 500 到 1000 字，必須包含真實 wiki 或社群資料超連結。文件模式由電子雞 persona、knowledge、intent mode、遊戲參數與 A2A 對話結果共同決定。
